# Тир-листы

Тир-лист — персональный список приоритетных объектов. Используется для формирования маркетингового пула совместно с AI-скорингом.

## Два уровня

### Тир-лист листинг-агента

Каждый агент формирует свой тир-лист из собственных объектов — те, которые считает наиболее перспективными для продажи и рекомендует для усиленного продвижения.

### Тир-лист тим-лида

Тим-лид формирует свой тир-лист независимо от агентских. Он видит все объекты в базе и выбирает приоритетные на основании общей картины: рыночной ситуации, баланса по районам, стратегических целей.

!!! note "Независимость"
    Тир-листы агентов и тим-лида не зависят друг от друга. Тим-лид не редактирует тир-листы агентов и наоборот.

## Лимиты

| Роль | Минимум | Максимум |
|------|---------|----------|
| Листинг-агент | 5 | 15 |
| Тим-лид | 20 | 50 |

Значения настраиваются в конфигурации модуля.

## Как работать с тир-листом

1. Откройте меню **Объекты → Тир-лист**
2. Вы увидите только свой тир-лист (тим-лид видит все)
3. Добавьте объект, укажите приоритет (перетаскиванием или числом)
4. При необходимости добавьте комментарий — почему объект в приоритете

## Приоритет

Приоритет определяет порядок объектов в тир-листе. **1 = высший приоритет**. Можно менять перетаскиванием строк в списке.

## Автоматическая валидация

Система проверяет тир-листы:

**Актуальность объектов:**

- Объекты со статусом «продан», «снят», «неактуален» помечаются
- Владелец тир-листа получает уведомление о необходимости замены

**Соответствие лимитам:**

- Ниже минимума — уведомление: «Добавьте объекты в тир-лист»
- Выше максимума — уведомление: «Сократите тир-лист»

Валидация срабатывает при обновлении тир-листа и еженедельно.

## Права доступа

- **Агент** — видит и редактирует только свой тир-лист
- **Тим-лид** — видит свой тир-лист + тир-листы всех агентов (только просмотр)
