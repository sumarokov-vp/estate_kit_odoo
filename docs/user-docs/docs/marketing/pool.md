# Маркетинговый пул

Маркетинговый пул — объекты, которые активно продвигаются на рекламных каналах. Размер пула настраивается (по умолчанию 100). Если подходящих объектов меньше — в пуле будет столько, сколько набралось.

## Marketing Pool Score (MPS)

Для каждого объекта рассчитывается **Marketing Pool Score** — комбинированный балл от 0 до 10, учитывающий два фактора:

- **AI-скоринг** (вес 60%) — автоматическая оценка цены, качества и карточки
- **Тир-листы** (вес 40%) — экспертная оценка агентов и тим-лидов

MPS отображается в списке объектов (колонка **MPS**) и на вкладке **Маркетинг** карточки объекта:

- 🟢 — объект соответствует порогу включения в пул
- 🟡 — в зоне гистерезиса (остаётся в пуле, если уже включён)
- 🔴 — ниже порога

## Расчёт пула

Для запуска расчёта:

1. Перейдите в **Недвижимость → Объекты → Рассчитать пул**
2. Расчёт запускается в фоне — вы получите уведомление
3. Прогресс можно отслеживать в **Объекты → Логи**

Расчёт выполняется в два этапа:

1. **AI-скоринг** — для объектов без свежего скоринга (старше 14 дней) запрашивается оценка через Anthropic Claude API
2. **MPS** — рассчитывается комбинированный балл с учётом тир-листов

## Правила включения

Объект включается в пул при выполнении **всех** условий:

- Статус объекта активен (не продан, не снят, не в архиве)
- Средний AI-скоринг ≥ минимального порога (по умолчанию 4.0)
- MPS ≥ порога включения (по умолчанию 7.0)
- В пуле есть свободные места

!!! info "Минимальный скоринг"
    Объект с низким AI-скорингом не попадает в пул, даже если тир-листы компенсируют балл. Тир-листы повышают приоритет *качественных* объектов, но не вытягивают слабые.

## Правила исключения

Объект исключается из пула при выполнении **любого** из условий:

- Статус стал неактивным (продан, снят, архив)
- MPS упал ниже порога исключения (по умолчанию 4.0)
- Средний AI-скоринг упал ниже минимума

При исключении:

- Тег **Marketing Pool** снимается
- Все активные размещения переводятся в статус «Снято»
- На объекте появляется комментарий с причиной

### Защита от исключения

Объект **не исключается** автоматически, если:

- Он в тир-листе тим-лида с высоким приоритетом (≤ 5)
- Есть активные размещения, генерирующие лиды

В этих случаях менеджер получает уведомление для ручного решения.

## Гистерезис

Порог включения (7.0) выше порога исключения (4.0). Объект в зоне между ними остаётся в пуле, если уже был включён, но не включается заново. Это предотвращает частое включение/исключение при колебаниях скоринга.

## Тег «Marketing Pool»

Объект в пуле отмечен тегом **Marketing Pool**. В списке объектов доступен фильтр **Маркетинг-пул** для быстрого просмотра состава пула.

## Автоматическая ротация

Еженедельно cron-задача пересчитывает MPS и обновляет состав пула автоматически.

## Контроль актуальности

Ежемесячно система создаёт задачи на обзвон по каждому объекту в пуле:

- Объект ещё в продаже?
- Изменилась ли цена?
- Есть ли обновления (ремонт, новые фото)?

Результат обзвона: **Неактуален** → снимается с пула, **Обновлён** → пересчитывается скоринг, **Без изменений** → остаётся.

## Настройки

Все параметры пула настраиваются в **Настройки → Estate Kit → Маркетинговый пул**:

| Параметр | По умолчанию | Описание |
|----------|:---:|---------|
| Размер пула | 100 | Максимальное количество объектов в пуле |
| Мин. средний скоринг | 4.0 | Минимальный AI-скоринг для попадания в пул |
| Порог включения | 7.0 | Минимальный MPS для автоматического включения |
| Порог исключения | 4.0 | MPS ниже которого объект исключается |
| Вес скоринга | 0.6 | Вес AI-скоринга в формуле MPS |
| Вес тир-листов | 0.4 | Вес тир-листов в формуле MPS |

!!! warning "Сумма весов"
    Вес скоринга + вес тир-листов должны равняться 1.0.

## Логи

Все шаги расчёта пула логируются в **Объекты → Логи** (категория `marketing_pool`). В логах отображается:

- Начало расчёта с параметрами
- Статус AI-скоринга (сколько свежих, сколько пересчитано)
- Итог: сколько обработано, сколько прошло в пул, сколько отклонено и почему
