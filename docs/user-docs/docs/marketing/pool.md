# Маркетинговый пул

Маркетинговый пул — это ~100 объектов, которые активно продвигаются на рекламных каналах. Объекты ротируются: неэффективные и неактуальные снимаются, добавляются новые.

## Как объект попадает в пул

Объект попадает в пул на основании двух факторов:

- **AI-скоринг** — автоматическая оценка конкурентности цены, качества контента и локации
- **Тир-листы** — экспертная оценка агентов и тим-лидов

AI-скоринг даёт объективную базу, тир-листы — экспертную корректировку.

## Тег «Marketing Pool»

Объект, включённый в пул, получает тег **Marketing Pool** (отображается в шапке карточки). По этому тегу можно фильтровать объекты в списке.

## Автоматическая ротация

Система **еженедельно** пересчитывает состав пула:

| Событие | Действие системы |
|---------|-----------------|
| Скоринг объекта упал ниже порога | Вывод из пула, размещения снимаются, уведомление менеджеру |
| Объект продан / снят с продажи | Автоматический вывод из пула |
| Новый объект с высоким скорингом | Уведомление менеджеру о кандидате в пул |
| Тир-лист повысил приоритет | Объект поднимается в очереди на включение |
| Тир-лист понизил приоритет | Может привести к выводу из пула при следующем пересчёте |

!!! info "Роль менеджера"
    Менеджер получает уведомления и выполняет физические действия (публикация, снятие объявлений), но решения о составе пула принимает система.

## Контроль актуальности

Ежемесячно система создаёт задачи на обзвон по каждому объекту в пуле:

- Объект ещё в продаже?
- Изменилась ли цена?
- Есть ли обновления (ремонт, новые фото)?

Сотрудник фиксирует результат в системе, после чего:

- **Неактуален** — снимается с пула, размещения закрываются
- **Обновлён** — карточка корректируется, пересчитывается скоринг
- **Без изменений** — остаётся в пуле
